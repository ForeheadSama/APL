How to Write Instructions (Statements)

Each instruction (or "statement") must end with a period (.).

DISPLAY "Welcome to APBL".

Declaring Information (Variable Setup)

Before you can use data like names, numbers, or dates, you must declare what type of data it is.

To declare a string (text):

string username.

To declare a number:

int ticketCount.

To declare a date:

date eventDate.

Making Decisions (If-Statements)

You can tell the program to do something only if a condition is true.

Basic "if" structure:

if (ticketCount > 5) [ DISPLAY "You have a lot of tickets". ].

"If...else" structure (for two possible paths):

if (ticketCount > 5) [ DISPLAY "Many tickets booked". ] else [ DISPLAY "Few tickets booked". ].

You can compare:

  • Two numbers (e.g., ticketCount > 5)
  • Two words or phrases (e.g., "VIP" == userType)
  • Dates (e.g., eventDate != "March 20, 2025")

Booking Tickets

To book tickets, use this format:

BOOK 2 TICKETS FOR "John Doe" ON "April 15, 2025" FOR "Concert X".

You can also use saved variables:

BOOK ticketCount TICKETS FOR username ON eventDate FOR eventName.

Cancelling Tickets

To cancel tickets:

CANCEL 1 TICKET FOR "John Doe" EVENT "Concert X".

Listing Events

You can list events by date:

LIST "Concert X" ON "April 15, 2025".

Or just list the event:

LIST "Concert X".

Checking Availability or Price

To check if seats are available or see the price:

CHECK AVAILABILITY FOR "Concert X" ON "April 15, 2025". CHECK PRICE FOR "Concert X" ON "April 15, 2025".

Making Payments

To pay for an event:

Displaying Messages

To show a message:

DISPLAY "Booking successful".

Or show the value of something you saved earlier:

DISPLAY username.

Accepting Input

To let a user type something in:

ACCEPT username.

This will let the user enter a value, like their name.

Adding Comments (Notes)

If you want to add notes in your code that the computer should ignore, start the line with $$.

$$ This is a comment explaining the next line DISPLAY "Thank you for using APBL".
